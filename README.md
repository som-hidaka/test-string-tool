# 入力フォームテスト文字列ツール マニュアル

## 概要

入力フォームのテストに使う文字列をブラウザ上で素早くコピーできるツールです。
テストしたいフィールドを選ぶだけで、そのフィールドに適したテスト文字列が一覧表示されます。

---

## 使い方

### STEP 1 ― テストしたいフィールドを選ぶ

画面上部のボタンからテスト対象のフィールド種別を選択します。

| フィールド | 説明 |
|-----------|------|
| テキスト | 一般的なテキスト入力欄 |
| メール | メールアドレス入力欄 |
| 電話番号 | 電話番号入力欄 |
| 数値 | 数値入力欄 |
| パスワード | パスワード入力欄 |
| URL | URL入力欄 |
| 検索 | 検索ワード入力欄 |
| すべて | 全フィールド共通の文字列を表示 |

### STEP 2 ― 境界値の条件を入力する（任意）

そのシステムの仕様に合わせた境界値を自動生成できます。

1. 最小文字数・最大文字数を入力する
2. それぞれ「半角」「全角」を選択する
3. 「生成」ボタンをクリック

**生成される文字列の例（最大文字数：100・半角の場合）**

| 文字列 | 内容 |
|--------|------|
| 最大-1（99文字） | 最大文字数を下回る |
| 最大（100文字） | 最大文字数ちょうど |
| 最大+1（101文字） | 最大文字数を超過 |

- 半角は `a`、全角は `あ` で文字列を生成します
- 「クリア」で生成した境界値を削除できます

### STEP 3 ― カテゴリで絞り込む（任意）

カテゴリボタンをクリックすると、特定カテゴリの文字列だけに絞り込めます。

| カテゴリ | 内容 |
|---------|------|
| 境界値 | 空文字・最小・最大付近の文字数など |
| 特殊文字 | 改行・タブ・記号・クォートなど |
| SQLインジェクション | SQLiの典型的なパターン |
| XSS | XSSの典型的なパターン |
| 日本語・全角 | 日本語・全角・半角カタカナなど |
| 絵文字 | 絵文字・サロゲートペア・ZWJシーケンスなど |
| Unicode | ゼロ幅スペース・RTL制御文字・ホモグリフなど |

### 文字列のコピー

各文字列の右側にある「コピー」ボタンをクリックするとクリップボードにコピーされます。
そのままテスト対象のフォームに貼り付けて使用してください。

---

## カテゴリ別テスト観点

### 境界値
入力制限（最小・最大文字数）の前後を確認するためのテストです。システムの仕様に合わせてSTEP 2で値を入力して使用してください。

### 特殊文字
改行・タブ・クォートなど、システムが正しく処理できるかを確認します。特にデータベース保存やHTML出力を行う箇所で重要です。

### SQLインジェクション
入力値がSQLクエリに組み込まれる箇所で、不正なSQL文が実行されないかを確認します。

### XSS
入力値がHTML上に出力される箇所で、スクリプトが実行されないかを確認します。

### 日本語・全角
全角文字・日本語・環境依存文字が正しく処理されるかを確認します。文字コードの問題や文字数カウントのズレが検出できます。

### 絵文字
絵文字や複合絵文字（ZWJシーケンス）が文字化けや切り捨てなく保存・表示されるかを確認します。

### Unicode
見えない文字（ゼロ幅スペース）や右から左への制御文字など、視覚的に確認しづらい文字の処理を確認します。

---

## 注意事項

- SQLインジェクション・XSSの文字列はセキュリティテスト用途のものです。テスト環境でのみ使用してください
- コピーした文字列はそのままの状態でフォームに貼り付けて使用します。エンコードなどの加工は不要です
